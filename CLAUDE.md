# CLAUDE.md

## セキュリティ

### シークレットをコンテキストに載せない(最重要)

- **目的ベースの禁止**: `.env`、`.env.local`、`.env.*` など環境変数ファイルの内容、および環境変数の実値を、**いかなる経路でも**コンテキスト・ログ・出力に載せないこと。手段を問わない(Read / Bash / Grep / Glob での読み取りに限らず、以下も禁止):
  - `node -e "console.log(process.env)"` や `printenv`、`Get-ChildItem env:` 等のプロセス環境変数ダンプ
  - dotenv を読み込むスクリプト・テストの実行結果に環境変数値を出力させること(値を echo/console.log するコードの実行)
  - `git diff` やエディタ系コマンド経由で .env 系ファイルの内容を表示すること
- 環境変数の**キー名**への言及は可。**値**は一切出力しない。
- 環境変数の値が必要な場合は、ファイルを開かずにユーザーへ確認する。
- 設定のサンプルが必要な場合は `env.defaults.json` など、秘密情報を含まないファイルのみを参照する。

## 破壊的操作の安全策(最重要)

**背景**: 2026-07-07、`adversarial-review` ワークフロー内のサブエージェントが、レビュー対象と無関係な未追跡ファイル(`CLAUDE.md`含む)を `rm -f`/`-rf` で削除する事故が発生した。多くは復元不能だった。この規律は自分(メインの対話)だけでなく、Task/Agent/Workflowツールで起動する**すべてのサブエージェント**に対しても、プロンプトで明示的に指示すること。

- **`rm -rf` を無条件にそのまま打たない**: 対象パスをその場で決め打ちして再帰・強制削除するコマンドをいきなり実行しない。
- **削除前に必ず `ls`(または `ls -la` / `find`)で対象を確認する習慣を徹底する**: 何が・何件消えるかを事前に目視確認してから削除に進む。ワイルドカードや `-r` を使う場合は特に。
- **確認付きの削除を使う**: `rm -rfi`(1件ずつ確認)、またはそれに相当する対話的確認を経由する。非対話環境で確認プロンプトが機能しない場合は、削除前に対象一覧を出力してユーザーに提示してから実行する。
- **重要な削除はゴミ箱コマンドを優先する**: `trash-cli`、`gvfs-trash`、macOSの `trash` コマンド等、OSのゴミ箱を経由する削除手段が使える環境ではそちらを優先し、直接 `rm` で完全削除しない。
- **スクリプト内では安全策を入れる**: 削除処理を書くシェルスクリプトには `set -u`(未定義変数参照をエラーに)、対象パスの存在確認、危険なパス(空文字列・ルート直下等)へのガードを必須にする。
