# SAFE MAGAZINE コンテンツ更新

通学路安全ニュース・SAFE MAGAZINE のコンテンツを更新するワークフローを実行します。

---

## 編集ミッション（すべての判断の最上位基準）

SAFE MAGAZINE は「ニュースの要約置き場」ではなく、**保護者の意思決定と安心感に直結する定期購読メディア**である。
すべての記事は公開前に次の3条件を自問し、1つでも満たさなければ書き直す:

| 条件 | 保護者の心の声 | 記事側の実装 |
|------|--------------|-------------|
| **納得** (Trust) | 「この編集部は信頼できる」 | 出典の格付け・データ時点の明記・断定と推測の書き分け |
| **満足** (Value) | 「読んでよかった。今日から使える」 | 危険情報1つにつき「今すぐできる行動」を必ず1つ以上セット |
| **再訪** (Habit) | 「また見に来よう」 | 続報予告・定点観測・季節連動・アプリ機能への導線 |

### 習慣化の設計原理（Hook Model の適用）

- **トリガー**: 年間編集カレンダー（後述）で「次に保護者に来るリスク」を先回りして配信する。入学・新学期・長期休みなど実生活のイベントに同期させる
- **行動**: 冒頭3行のリードだけで「わが家に関係あるか」「読むと何ができるようになるか」が分かる構成にし、開いた瞬間の離脱を防ぐ
- **可変報酬**: 定点観測データの更新、過去記事の「その後」、読者ヒヤリハットの紹介——「前回と違う何か」が毎回ある状態を作る
- **投資**: チェックリスト実施・アプリでの通学路確認・危険箇所報告へ誘導する。読者が行動するほど次の記事が「自分ごと」になる

## 記事テンプレート（6種類から必ず選択）

記事化の際は、目的に応じて以下のいずれかのテンプレートを選び、記事冒頭のコメントまたは作業メモにテンプレート名を記録すること。

### T1: 速報型 (news-flash)
- **用途**: 法改正・制度施行・公的発表の直後
- **鉄則**: 冒頭3行で「いつから」「誰に関係するか」「何をすべきか」を言い切る
- **構成**: リード3行 → 制度の概要 → わが家への影響（学年別） → 今すぐやること → 今後のスケジュール → 出典
- **既存例**: 自転車青切符記事

### T2: 統計解説型 (data-explainer)
- **用途**: 公的統計の公表・年次更新
- **鉄則**: 数字を必ず「わが家の場合」に翻訳する（例: 「小1は小6の2.5倍」→「入学後の最初の2ヶ月が勝負」）。最重要の数字は記事全体で1つに絞る
- **構成**: 最重要の数字1つ → なぜそうなるのか → 学年別に何を意味するか → 今すぐできる行動 → 出典（調査年・集計期間を明記）
- **既存例**: 魔の7歳記事

### T3: 地域事例型 (local-case)
- **用途**: 自治体・地域の先進的な取り組み紹介
- **鉄則**: 「あなたの自治体で確認する方法」セクションを必ず入れる（確認先の種類・検索キーワード・問い合わせ窓口の見つけ方）。よその話で終わらせない
- **既存例**: 加古川市見守りカメラ記事

### T4: シリーズ追跡型 (follow-up)
- **用途**: 過去記事の続報（施行後の実態・統計の年次更新・モデル事業の進捗）
- **鉄則**: 冒頭に「前回のおさらい」3行と前回記事へのリンクを必ず置く。tags に `シリーズ:{シリーズ名}` を付与する
- **候補管理**: 「施行日が未来の制度」「毎年更新される統計」「進捗が発表されるモデル事業」を扱った記事は、公開時点で続報候補になる

### T5: 実践ガイド型 (action-guide)
- **用途**: チェックリスト・声かけ・練習方法などの実用コンテンツ
- **鉄則**: 「保存版」であることを明記し、印刷・スクリーンショットで繰り返し使える構成にする。チェックリストは具体的な行動単位で書く（「気をつける」ではなく「横断歩道の手前で止まって左右を見る練習を3回する」）
- **既存例**: ながら見守り記事

### T6: 季節連動型 (seasonal)
- **用途**: 年間編集カレンダーに基づく先回り情報
- **鉄則**: 「なぜ今この話をするのか」の根拠となる統計・傾向を必ず1つ以上示す（当年データをリサーチで確認してから記事化する）

## 年間編集カレンダー（配信テーマの先回り計画）

各月のテーマは「リサーチで当年の裏付けデータを確認したうえで」記事化する。

| 時期 | 重点テーマ | ねらい |
|------|-----------|--------|
| 2〜3月 | 入学準備・通学路の親子下見 | 4月の新1年生リスクへの先回り |
| 4月 | 新学期・春の全国交通安全運動 | 運動期間・取り締まり強化と同期 |
| 5月 | 新1年生の事故ピーク（GW明け） | 小1歩行中事故は5月に急増する傾向 |
| 6月 | 梅雨の視界不良・傘さし歩行 | 雨天時の視認性・行動の変化 |
| 7月 | 夏休み前・行動範囲の拡大 | 自由行動時間の増加、水辺の事故 |
| 9月 | 新学期・生活リズムの立て直し | 長期休み明けの注意力低下 |
| 10〜11月 | 薄暮時間帯の事故・反射材 | 日没時刻の急変期 |
| 12〜1月 | 積雪凍結・年末年始の交通量 | 冬季路面と帰省期の交通環境変化 |

## 「また来たくなる」仕組み（記事作成時に最低3つ組み込む）

1. **次回予告**: 記事末尾に「次回は○○を取り上げる予定です」を入れる（実際に予定しているテーマのみ）
2. **シリーズタグと前後リンク**: `シリーズ:{名前}` タグで束ね、前後の記事に相互リンク
3. **定点観測企画**: 同じ指標（事故統計・対策完了率など）を定期的に追う「月次ウォッチ」枠
4. **続報追跡リスト**: 施行待ちの制度・進行中のモデル事業を記録し、動きがあれば T4 で続報化
5. **学年別ターゲットバッジ**: tags に `対象:新1年生` `対象:低学年` `対象:高学年` `対象:中高生` を付与し、読者が「うちの子の記事」を見つけやすくする
6. **「あなたの自治体で確認」定型セクション**: 全国ニュースを地域の行動に変換する
7. **エビデンスレベル表示**: 出典に格付けを添える（後述の表記ルール）
8. **データ時点表示**: すべての統計に「（2025年○月時点）」等を明記。更新時は本文冒頭に更新履歴を記す
9. **保存版チェックリスト**: T5 記事は保存・再訪の起点になる
10. **今週の声かけテンプレート**: 「今週、子どもに伝えたい一言」をkeyPointsまたは本文末尾に入れる
11. **アプリ連動CTA**: 記事末尾で mapsefe の機能（マップで通学路チェック・危険箇所報告）へ誘導する。記事とアプリの往復が習慣を作る
12. **読者ヒヤリハット紹介**: アプリの報告データを匿名集計して「みんなの気づき」として紹介する（個人・場所が特定できる情報は絶対に出さない）

## 必須ルール（厳守）

### ルール1: 事実のみを記載する
- 実際に検索・取得して確認できた情報のみを使用すること
- 架空・推測・要約の捏造は禁止
- 出典URLが実在し、内容が確認できるもののみを記事化する
- 不確かな事故・事件の詳細は記事化しない（政策・法改正など公式情報源があるものを優先）
- **統計は「調査主体・調査年・集計期間」をセットで記載する**。時点が書けない数字は使わない

### ルール2: 重複記事を作成しない
- 記事作成前に必ず `content/safe-magazine/articles/` と `lib/safe-magazine.ts` の既存記事を確認すること
- 既存記事と同一・類似トピックは記事化しない
- 類似判定基準: 同じ法律、同じ事件、同じ取り組みを扱っている場合は重複とみなす
- **例外＝続報**: 同じトピックでも「新しい事実（施行後の実態・更新された統計・進捗発表）」があれば T4 シリーズ追跡型として記事化してよい。その場合は前回記事へのリンクと差分の明示が必須

### ルール3: 不安と行動のセット原則
- 危険・リスクを伝える記述1つにつき、「今すぐできる行動」を必ず1つ以上セットで提示する
- 行動の提示なしに不安だけを残す記事は公開しない
- 恐怖訴求で釣るタイトル・サムネイルは禁止

### ルール4: 「わが家に翻訳」原則
- 全国統計・制度の話は、必ず「読者の家庭では何を意味するか」まで翻訳する
- 学年別・状況別に影響が異なる場合は書き分ける（例: 16歳未満は青切符の対象外）

### ルール5: エビデンスの見せ方
- 出典には信頼度の格付けを添える:
  - `[公的統計]` 警察庁・国交省・内閣府などの一次データ
  - `[公式発表]` 自治体・省庁の発表資料
  - `[報道]` 大手メディアの一次報道
  - `[専門機関]` 交通安全協会・損保協会・警備会社など
- 断定できる事実と、傾向・見解は文体で区別する（「〜です」vs「〜という傾向が示されています」）

### ルール6: 継続性の設計
- すべての記事に関連記事リンクを1つ以上入れる
- 続報候補になる記事（未来の施行日・年次統計・進行中事業）は、作業レポートに「続報候補」として記録する

## 執筆ガイドライン

### タイトルの公式
`【いつ/誰に】 + 具体的な数字か固有名詞 + 読者ベネフィット` — 32文字目安、最大40文字

- 良い例: 「【4月から施行】自転車も『青切符』の対象に——中高生の通学に何が変わる？」
- 悪い例: 「自転車のルールが変わります」（いつ・誰に・何がが不明）

### excerpt の公式（120〜160字）
`事実1文 + わが家への意味1文 + この記事で得られるもの1文`

### keyPoints の公式（必ず3つ）
1. **最重要ファクト**（数字入り）
2. **読者への意味**（わが家に翻訳した一文）
3. **今すぐできる行動**（具体的な動作）

### 本文の共通骨格
1. リード3行: 「何が起きた/分かった」「わが家にどう関係」「読むと何ができる」
2. 本文: 事実 → 解釈 → 行動 の順で展開
3. 「今すぐできること」セクション（必須）
4. 「あなたの自治体で確認する方法」（全国ニュースの場合）
5. 出典（格付け・データ時点つき）
6. 関連記事リンク・次回予告

### トーン＆マナー
- 不安を煽りすぎない。危険の指摘は必ず行動の提案で受ける
- 専門用語には解説を添える
- 保護者を責めない（「〜すべきなのにできていない」ではなく「〜すると守りやすくなる」）

## データ構造の拡張規約（後方互換）

`lib/safe-magazine.ts` の `SafeMagazineArticle` 型を壊さず、**tags の命名規約**で拡張情報を表現する:

- `シリーズ:{名前}` — シリーズ追跡型の束ね（例: `シリーズ:30km/h規制ウォッチ`）
- `対象:新1年生` `対象:低学年` `対象:高学年` `対象:中高生` — 学年ターゲット
- `地域:{都道府県}` — 地域事例型の対象地域（例: `地域:兵庫県`）

型フィールドを追加する場合は optional にし、既存記事とテストを壊さないこと。

## チーム構成

1. **ニュースリサーチャー** (`.claude/agents/news-researcher.md`) — カテゴリ別の最新情報収集
2. **ファクトチェッカー** (`.claude/agents/fact-checker.md`) — 出典検証・クロスチェック・APPROVED判定
3. **記事ライター** (`.claude/agents/article-writer.md`) — 本ファイルの編集ミッションとテンプレートに従い執筆
4. **ビジュアルデザイナー** (`.claude/agents/visual-designer.md`) — サムネイル・図解仕様と画像プロンプト作成
5. **画像ジェネレーター** — `scripts/generate-safe-magazine-images.ts` で画像生成（モデルは `GEMINI_IMAGE_MODEL` 環境変数、既定: `gemini-3.1-flash-lite-image`）
6. **SEOスペシャリスト** (`.claude/agents/seo-specialist.md`) — メタデータ・構造化データ・内部リンク

チーム全体のオーケストレーションは `/run-safe-magazine-team` を参照。

## ワークフロー

$ARGUMENTS に指定されたカテゴリでリサーチを開始します。カテゴリ未指定の場合は全カテゴリ＋続報チェックを対象とします。

利用可能なカテゴリ: `accident-news` / `volunteer-activity` / `safety-tips` / `danger-ranking` / `policy-update`

### Step 0: 既存記事の確認と続報候補の抽出（必須）
1. `content/safe-magazine/articles/` と `lib/safe-magazine.ts` の既存記事を確認し、重複トピックを除外する
2. 既存記事のうち「施行日が未来の制度」「年次更新される統計」「進行中のモデル事業」を扱ったものを洗い出し、続報化すべきものがないか判断する（例: 30km/h規制の施行後実態、青切符の運用状況、ゾーン30プラスの進捗）

### Step 1: ニュースリサーチ
年間編集カレンダーで「今の時期」に該当するテーマを優先しつつ、カテゴリ別キーワードで検索する:

- 事故: 「通学路 事故」「登下校 児童 事故」＋当年
- 見守り: 「通学路 見守り ボランティア」「スクールガード」
- 防犯: 「子ども 防犯 対策」「不審者 対策 通学路」
- 危険箇所: 「交差点 危険度」「通学路 危険箇所 マップ」
- 施策: 「通学路 安全対策 自治体」「スクールゾーン 規制」

リサーチ結果を `content/safe-magazine/research/YYYY-MM-DD-research.json` に保存する。

### Step 2: 記事作成
ファクトチェック済みの情報から、テンプレート（T1〜T6）を選んで執筆する。
Markdown を `content/safe-magazine/articles/YYYY-MM-DD-{slug}.md` に保存する。

### Step 3: 画像生成
`scripts/generate-safe-magazine-images.ts` の `ARTICLE_IMAGES` 配列に新記事の `ImageConfig` を追記してから実行する:

```bash
npx tsx scripts/generate-safe-magazine-images.ts
```

出力先: `public/images/safe-magazine/thumbnails/{slug}.png`、`public/images/safe-magazine/articles/{slug}/{image-id}.png`

画像プロンプトの日本向け要素: 黄色い通学帽・ランドセル・日本の住宅街・横断歩道・グリーンベルト・ガードレール・スクールゾーン標識・見守りボランティア。アニメ/マンガ風の温かみのあるイラスト。実在人物の顔・実在の事故現場の再現は禁止。

### Step 4: 公開統合（これを忘れると記事はアプリに表示されない）
1. `lib/safe-magazine.ts` の `ARTICLES` 配列に新記事オブジェクトを追加する
2. ランディングに出す場合は `lib/landing-safe-magazine-preview.ts` を更新する

### Step 5: 検証ゲート（緑になるまで完了と言わない）
```bash
npx vitest run tests/unit/lib/safe-magazine-release-readiness.test.ts
npx tsc --noEmit
```
その後キャッシュクリアと表示確認:
```bash
rm -rf .next/cache
npm run dev
```
ブラウザでハードリロード（Ctrl+Shift+R）し、一覧・詳細・サムネイルを確認する。

### Step 6: 作業レポート
- 作成記事（テンプレート種別つき）／続報候補リストの更新／組み込んだ「また来たくなる仕組み」の一覧／次回更新の推奨テーマ（年間カレンダー準拠）を報告する

## 効果測定の指標

| 区分 | 指標 | 見るもの |
|------|------|---------|
| 定量 | 7日以内の再訪率 | 習慣化の進み具合 |
| 定量 | 記事読了率・スクロール深度 | リード3行と本文構成の質 |
| 定量 | シリーズ2本目の閲覧率 | 続報・シリーズ設計の効果 |
| 定量 | アプリ連動CTAのクリック率 | 記事→アプリ往復の成立 |
| 定量 | チェックリスト到達率 | 「満足」条件の達成度 |
| 定性 | 「読んでよかった」フィードバック | 納得・満足の直接評価 |
| 定性 | 保護者ヒアリング（信頼できるか・また読みたいか） | 3条件の総合評価 |
