# 小学生の親向け 通学路安全確認アプリ UXリサーチまとめ

## 概要

本ドキュメントは、`小学生の親` をターゲットにした `子どもの通学路の安全確認アプリ` について、`100人の仮想ユーザー` をシミュレーションして整理したUXリサーチ結果です。

注意:
- 実地インタビューではなく、仮想ユーザー分析です
- リポジトリから読み取れた範囲の機能前提を含みます
- そのため、機能評価は方向性の確認に使い、最終判断は実ユーザー検証で補完してください

## 調査前提

- 主対象ユーザー: 小学生の親
- 中心価値: 子どもの通学路の安全確認
- 想定される主機能:
  - 通学路の登録
  - 地図上でのルート表示
  - 危険箇所や安全情報の確認
  - 家族や学校と共有したくなる安全確認体験

## ユーザータイプ

1. 不安感が強い慎重派
事故や不審者情報に敏感で、根拠つきの安全情報を求める。

2. 忙しい時短派
朝の短時間で結論だけを知りたい。操作量が多いと離脱しやすい。

3. 地図が苦手な非デジタル派
地図操作や設定が苦手で、テキスト中心の分かりやすさを重視する。

4. 比較検討派
複数ルートを安全性、距離、時間で比較して納得感ある判断をしたい。

5. 連携重視派
家族、学校、地域との共有や連携に価値を感じる。

## 第一印象

### ポジティブ

- 子どもの通学路を地図で確認できるのは安心感がある
- 危険箇所が見えるなら実用性が高そう
- 毎朝の確認習慣に組み込みやすそう
- 家族で共通認識を持てそう
- 防犯と交通安全の両方に役立ちそう

### ネガティブ

- 最初に何をすればいいか分かりにくい
- 地図中心で少し難しく見える
- 安全判定の根拠が見えないと信用しづらい
- 情報が古いと逆に不安になる
- 入力や設定が多いと続かなそう

## 仮想ユーザーの声から見えた傾向

### よく出たポジティブ反応

- 安心感がある
- 子どもへの声かけに使えそう
- 家族共有に向いていそう
- 毎朝の確認に使えそう

### よく出たネガティブ反応

- 初回導線が弱い
- 地図が難しい
- 安全判定の根拠が不明
- 情報の鮮度が不安
- 朝使うには操作が重い

### よく出た要望

- 一目で安全か分かるホーム
- 危険箇所の理由説明
- ルート比較
- 家族共有
- 条件別リスク表示

## UX問題 TOP10

1. 初回起動時に、どう使えば安全確認できるか分かりにくい
2. 危険箇所の意味や危険度の基準が伝わりにくい
3. 地図中心で、忙しい親には情報取得に時間がかかる
4. ルート作成や編集の操作負荷が高い
5. スマホ画面で情報が重なり、見たい情報を絞り込みにくい
6. 安全かどうかの結論が一目で分からない
7. 親が見たい観点がまとまっていない
8. 更新日時や情報源が弱く、信頼性が揺らぐ
9. 子どもごとの管理がしにくい
10. 家族や学校との共有導線が弱い

## 欲しい機能

### 必須機能

1. 通学路ごとの安全スコア表示
2. 危険箇所の詳細と理由説明
3. 朝の時短確認用サマリー
4. 家族共有機能
5. 情報更新日時の明示

### あると嬉しい機能

1. 雨の日や暗い時間帯など条件別リスク表示
2. 学校推奨ルートとの比較
3. 地域の見守り情報や通報連携
4. 複数の子どものルート管理
5. 登下校前の通知リマインド

## ペルソナ

### 1. 佐藤 美咲

- 36歳
- 1年生の母
- 共働き
- 目的: 朝30秒で今日の通学路が安全か知りたい
- 課題: 地図操作や設定に時間をかけられない

### 2. 田中 恒一

- 42歳
- 3年生の父
- 慎重派
- 目的: 危険判定の根拠を見て納得して判断したい
- 課題: 曖昧な危険表示は信用しない

### 3. 山本 綾

- 39歳
- 2人の子の母
- 非デジタル寄り
- 目的: 迷わず使える画面で安全確認したい
- 課題: 設定や編集が複雑だと使えない

### 4. 鈴木 恒一

- 45歳
- 引っ越し直後の父
- 比較重視
- 目的: 複数ルートを比較して最も安全な道を選びたい
- 課題: 地図だけでは比較しにくい

### 5. 高橋 由紀

- 38歳
- PTA参加中の母
- 共有重視
- 目的: 家族、学校、地域で安全情報を共有したい
- 課題: 個人利用だけでは継続価値が弱い

## 離脱理由 TOP5

1. 情報更新が少なく、毎回見る価値を感じにくい
2. 操作が面倒で、既存の地図確認に戻ってしまう
3. 危険判定の根拠が弱く信用できない
4. 欲しい情報が地図の中に埋もれて探しにくい
5. 家族や学校と共有できず、単独利用で終わる

## 継続利用理由 TOP5

1. 毎日すぐに安全状況を確認できる
2. 危険箇所が具体的で子どもへの声かけに使える
3. ルート比較で納得感ある判断ができる
4. 家族間で同じ情報を共有できる
5. 更新頻度が高く、信頼できる

## 開発優先順位

1. 一目で安全判断できるホームサマリー
2. 危険箇所の根拠、更新日時、情報ソースの可視化
3. ルート比較と安全スコアの分かりやすい提示
4. スマホ前提の地図UI整理と入力負荷の削減
5. 家族共有と学校連携の強化

## 改善チェックリスト

### P0: 先に着手すべき項目

- [ ] ホーム画面で `今日の通学路は安全 / 注意 / 要確認` を即表示する
- [ ] 初回起動時に、3ステップ以内で価値が分かるオンボーディングを入れる
- [ ] 危険箇所ごとに `何が危険か` `更新日時` `情報源` を表示する
- [ ] ルート登録を簡略化し、学校名や住所から初期ルートを自動生成できるようにする
- [ ] スマホで片手操作しやすいナビゲーションに整理する

### P1: 早めに改善したい項目

- [ ] 複数ルートの `安全性 / 距離 / 所要時間` 比較を提供する
- [ ] 危険箇所を一覧と地図の両方で確認できるようにする
- [ ] 家族共有機能を提供する
- [ ] 子どもごとに通学路を分けて管理できるようにする
- [ ] 雨天、暗い時間帯、交通量増加時などの条件別リスクを表示する

### P2: 価値を伸ばす項目

- [ ] 学校推奨ルートとの比較を表示する
- [ ] 保護者や地域から危険箇所を報告できるようにする
- [ ] 登校前の通知や注意喚起通知を実装する
- [ ] PTAや学校向けの共有ビューを用意する
- [ ] 登校班、集合場所、見守り地点の管理に対応する

## ひとことで言うと

このアプリに対して親が一番強く求めているのは、`地図の多機能さ` よりも `短時間で信頼できる安全判断ができること` です。
