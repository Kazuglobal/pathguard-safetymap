# PathGuardian プロダクト概要(再構築版)

## コンセプト

子どもの通学路の安全を、子ども・保護者・学校が一緒に守るための地域参加型プラットフォーム。「危険を見つける遊び」を通じて子ども自身が学び、その成果を家族と共有する、という一貫した体験設計を軸にしている。

## 技術スタック

- Next.js 16 (App Router) + React 19
- Mapbox GL(地図描画・POI検索)
- Supabase(認証・Postgres・Storage・RLS)
- Vitest(ユニット/コンポーネントテスト)、Playwright(レスポンシブ/E2E)

## 主要機能

1. **危険箇所レポート**(`/report`、`app/report/page.tsx` ほか)
   - 写真付きで危険箇所を投稿→承認後に地図へピン表示→カテゴリ別(交通危険/不審者・犯罪/災害危険/不審者情報/その他)に一覧化
   - PDF/PNG/JPEG形式でのレポート出力(`lib/report-generation/`)。教育版デザイン(星による危険度表示、子ども向け言い回し)
   - 画像は非公開Storageバケット+署名URLで配信(`lib/danger-report-image-access.ts`)

2. **きけんハンター**(`/safety-quest/hunter`)
   - AIが写真から危険を検出する「危険さがしの練習」ゲーム。専用ハンターAIによる判定
   - 「たんけんノート」デザイン言語(紙質感・Zen Maru Gothic・探検ノート風レイアウト、キャラクター「ルペ」)

3. **通学路管理**(`/routes`)
   - 家族で通学路を登録・管理。地図の初期表示は登録ルート中心

4. **ホーム / ニュースフィード**(`/landing`)
   - ヒヤリハット情報のデイリーハビット設計。地域(都道府県)フィルタ対応

5. **SAFE MAGAZINE / 学校ニュース**(`/safe-magazine`, `/school-route-news`)
   - 編集コンテンツ・日次更新の安全記事。Push通知連携

6. **活動**(`/mypage`)
   - プロフィール・ミッション・バッジ・リーダーボードによるゲーミフィケーション

7. **家族共有カード**(`lib/report-generation/family-share-card.ts`)
   - 危険報告の要点だけを抜き出し、家族にWeb Share API/ダウンロードで共有

8. **管理者ダッシュボード**(`/admin/dashboard`)
   - レポートのモデレーション・承認フロー

## 安全性に関する設計原則(このセッションで確認済みのもの)

- 不審者アラートはクライアント側判定ではなく、サーバ側LLM/Vision(`moderateSuspiciousAlertWithAi`)による実審査を経由する
- 危険レベル(1〜4)の表示は `lib/report-generation/danger-level-presentation.ts` に一元化(星表示、色覚多様性・モノクロ印刷対応)
- 偽データ・偽実績(ダミー店舗商品・偽キャンペーン等)は2026-07-04の親向けUI刷新で撤去済み、復活させない方針
- `.env` 等の環境変数の値をコンテキスト・ログ・出力に一切載せない(CLAUDE.mdの最優先ルール)
