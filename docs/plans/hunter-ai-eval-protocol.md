# きけんハンター AI 品質 評価プロトコル

専用ハンターAI(`lib/hunter/hunter-ai.ts`)の品質は2層で守る。

## 1. ゴールデン回帰(自動・CI内)

`tests/unit/lib/hunter/golden.test.ts` + `tests/unit/lib/hunter/golden/*.json`。

- 実写真→Gemini はライブ&非決定的なので CI には載せない。代わりに **代表的な「生AI出力」フィクスチャ**で後処理パイプライン(`validateHunterResponse → sanitizeDangerPoints → buildQuizItemsFromAi`)の出力を固定測定する。
- カバーする不変条件: 件数 cap、全画面/極小 bbox の除外、IoU 重複統合、severity×confidence ランキング、英語/空文言の KID_COPY 置換、kid ラベル正規化、安定ID、place 昇格の受け入れ基準(confidence≥0.7 かつ面積[0.02,0.35])。
- **プロンプトや sanitize を変えたら、まずこのテストが赤くなる**。意図した変更ならフィクスチャ/期待値を更新する。
- 実行: `npx vitest run tests/unit/lib/hunter/golden.test.ts`。

## 2. ライブ人手レビュー(手動・定期)

ゴールデンは「壊れていないこと」を守るが、「実写真で良いか」は人が見る。リリース前/プロンプト改修後に実施。

### 手順
1. 代表的な通学路写真を **N=10 枚以上**用意(住宅街の細道・見通しの悪い角・信号のない横断・路上駐車のある道・歩道のない道・明るい/暗い・縦/横 を混ぜる)。
2. 各写真で `/safety-quest/hunter` を実プレイ(マスク→ピン→解析)。
3. 下記チェック項目で 1 枚ずつ評価(○/△/×)。

### チェック項目
- **危険ポイントの妥当性**: 返ってきた点は「立ち止まって確かめるべき状況」か。普通の停車中の車・看板・自転車を“危険”にしていないか(的外れ正解の混入率)。
- **bbox の概ねの位置**: 当たり判定ゾーン(寛容)で、見えている危険を素直にタップして hit に到達できるか。縦長写真でズレないか。
- **取りこぼし**: 明らかな危険(死角・飛び出し地点・信号なし横断)を見落としていないか。0件(guide)に落ちる頻度。
- **文言の難易度**: whyDangerous/safeAction が 1〜2 年生語彙か。難語・英語ラベルの露出はないか。こわがらせ/断定はないか。
- **クイズの質**: 設問が写真連動か。誤答が「もっともらしい思い込み」になっているか(荒唐無稽でないか)。place 設問の正解ゾーンが妥当か。
- **事故連携**: 近隣の多い事故タイプ(care-card)と、出る危険ポイント/accidentLink が概ね整合するか。
- **フォールバック**: 暗所/通学路でない写真で guide(unusable) に落ち、「安全」と断定しないコピーが出るか。

### 記録
- スプレッドシート等に「写真ID・各項目の評価・自由コメント」を残す。的外れ正解率と取りこぼし率の傾向を追う。
- 悪化が見られたら、その写真の生AI出力を新しいゴールデンフィクスチャに追加して自動回帰へ昇格させる。
