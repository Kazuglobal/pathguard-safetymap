# Mobile Heatmap Button Stack Design

## Goal

モバイル地図で左上の `事故` ボタンが `ハザード・地図` ボタンの背後に隠れないようにする。

## Current Problem

- `ハザード・地図` は左上 `top-[calc(env(safe-area-inset-top,0px)+4.25rem)]` に表示される。
- `事故` も同じ左上帯に表示される。
- `事故` は `z-10`、`ハザード・地図` は `z-20` のため、スクリーンショットどおり `事故` が完全に隠れる。

## Chosen Approach

モバイル時のみ `事故` ボタンの縦位置を下げ、左上で 2 段に積む。

### Why

- 既存 UI の意味を変えない。
- オーバーレイ優先順位はそのまま維持できる。
- 修正範囲が `事故` 側レイアウトに閉じる。

## Non-Goals

- デスクトップ配置の変更
- ボタンの文言や見た目の再設計
- `ハザード・地図` ドロワーや `事故` ドロワーの機能変更

## Validation

- レイアウトヘルパーのモバイルクラスに、重ならない縦位置が入ることをテストで固定する。
- 既存のモバイル heatmap / floating controls テストを再実行する。
